DESIGN RULES & AI INSTRUCTIONS
Mandatory rules for Google AI Studio code generation. These override all default AI design assumptions.
Rule 1 — No emojis. All icons must use inline SVG (stroke-based, 16x16 or 24x24 viewBox). Never use unicode emoji characters, emoji fonts, or emoji shortcodes anywhere in the dashboard. Use the SVG patterns demonstrated in the Callouts and Buttons sections above.
Rule 2 — No hardcoded colors. Every color reference in component code must use a CSS custom property (e.g., var(--teal-700)). Never write a hex code directly in a component's inline style or class. The only permitted hex values are inside the :root token block.
Rule 3 — Font loading. Every page must include the Google Fonts import for Barlow Condensed + DM Sans + DM Mono. Do not substitute Inter, Roboto, Arial, or system-ui. Display numerics always use Barlow Condensed. Body text always uses DM Sans.
Rule 4 — Alamtri blob pattern. Any dark hero/header panel must include the two pseudo-element blobs: a gold radial-gradient at top-right (--gold-300 to --gold-500) and a subtle teal glow at bottom-left (--teal-400 at 18–20% opacity). These are the visual signature of the Alamtri brand.
Rule 5 — Table header color. All data table thead rows must use background: var(--teal-700) with white text. Never use grey or light backgrounds for table headers.
Rule 6 — Status color discipline. Green = On Track only. Amber = At Risk only. Red = Delayed / Critical only. Blue = Forecast only. Grey = Not Started only. Do not repurpose these colors for branding, decoration, or category coding. Party coding (EP/CC/PMC) uses the badge system defined in Section 4.
Rule 7 — Module structure. Every dashboard module page follows: Hero Dark panel (Section 12 pattern) → KPI tiles row (Section 3) → primary data component (chart/table/Gantt) → supplementary cards. The sidebar and topbar chrome are always present and fixed.
Rule 8 — CHPP2 domain vocabulary. Use the following terms exactly: "EP Contractor" (not "engineering contractor"), "Construction Contractor" (not "builder"), "Employer" (not "client"), "PMC / Employer's Engineer" (not "project manager"), "Taking-Over Certificate" (not "handover"), "Variation Order" (not "change order"), "CLSTK" for Phase 1 contract mechanism, "WIB" for all timestamps (Indonesia Western time, UTC+7).
