Executive Summary & Objectives

ℹ️
This document defines the complete end-to-end development workflow for the CHPP2 Project Dashboard, a web-based reporting and tracking platform for PT Maruwai Coal as the Employer. The dashboard consolidates all contractual, commercial, schedule, risk, and document control data across three governed parties: the Project Management Consultant / Employer's Engineer (PMC/EE), the EP Contractor, and the Construction Contractor.

🎯 Primary Objectives

Provide the Employer with a single-pane-of-glass view of all project activities, giving real-time visibility into progress, commercial exposure, risk profile, and vendor performance against the USD 89.2M EPC programme.

Enable structured exception management — surface problems early before they cascade into schedule or cost overruns in the remote Central Kalimantan environment.

📐 Design Principles

Employer-centric: All KPIs, alerts, and workflows reflect the Employer's obligations and rights under the split EP + Construction contract structure.

Excel/Sheets-native data: Zero new database infrastructure — all source data lives in structured Excel/Google Sheets that field teams already maintain via Microsoft Teams.

Progressive disclosure: Executive summary on landing page; drill-down to individual line items on demand.

📊 Dual-Bar Display

  • Row 1 (top, grey): Frozen baseline bar — never changes once baseline is locked
  • Row 2 (bottom, colour): Actual progress bar with % fill + forecast extension (dashed)
  • Both bars aligned to the same timeline axis for instant visual comparison
  • Colour coding: Green (on track) / Amber (SPI 0.85–0.99) / Red (<0.85 or missed date) / Blue dashed (forecast)
  • Progress fill within bar corresponds to % complete field from source sheet

◆ Milestone System

  • Diamond shape (◆) rendered at exact date position on timeline
  • Two diamonds per milestone: baseline date (grey outline) + actual/forecast date (filled colour)
  • States: Achieved (green ◆) / Pending on-track (amber ◆) / Overdue (red ◆ + flag)
  • Tooltip on hover: Baseline date, Forecast date, Variance (days), Responsible party
  • All 9 contract key milestones (from Employer Requirements §9.1) pre-loaded as fixed entries
  • Milestone-only view toggle: collapses all task bars, shows only diamonds on timeline

🔗 Dependencies & Critical Path

  • Finish-to-Start (FS), Start-to-Start (SS), Finish-to-Finish (FF) arrows rendered as SVG lines
  • Lag/lead time shown numerically on arrow (e.g., "+5d")
  • Critical path toggle: highlights entire chain in red when enabled
  • Float display: grey extension bar showing available slack days per task
  • Predecessor column in left panel links activities for easy reference

🌧 CHPP2-Specific Features

  • Wet season risk band (Dec–Feb) automatically highlighted in soft blue on timeline — warns when delivery/construction activities overlap
  • Barito River navigation constraint flag on any logistics activity falling in Dec–Feb window
  • CHPP2A vs CHPP2B parallel track view — side-by-side unit comparison
  • CLSTK Phase 1 / Phase 2 boundary marker on timeline (conversion point)
  • EP Contract vs Construction Contract colour-coded swim lanes

🔍 Filtering & Navigation

  • Filter by: Party (EP/CC/PMC/Employer), Discipline, Phase, WBS Level, Status (On Track/At Risk/Delayed)
  • Search by activity name (instant keyword highlight)
  • Collapse/expand WBS levels 1–4 individually or globally
  • Time-scale zoom: Weekly (near-term detail) / Monthly (standard view) / Quarterly / Yearly (full project overview)
  • Scroll-to-today button — always centres the view on current date
  • Drag-to-scroll timeline horizontally

📸 Baseline Snapshot Management

  • Baseline Rev A: initial contract baseline (frozen at contract award)
  • Baseline Rev B, C…: formally approved revised baselines after accepted EOTs
  • Snapshot comparison: overlay any two revisions to visualise schedule growth
  • Snapshot stored as dedicated tab in GANTT_Schedule_Master sheet (columns duplicated with revision suffix)
  • Variance column auto-calculated: Forecast Finish minus Baseline Finish (days)
  • Baseline lock: once Rev A is set, baseline columns are sheet-protected

📤 Export Capabilities

  • PDF export: A3 landscape, full chart with legend — suitable for progress meeting packs
  • PNG export: high-resolution snapshot for email/presentation embedding
  • Excel export: structured data dump of current view (filtered activities only)
  • Print-friendly CSS mode for direct browser printing
  • Share: generate a read-only public link (time-expiring) for board/external stakeholder review

⚙️ Technical Implementation

  • Rendering library: DHTMLX Gantt (open-source) or Frappe Gantt (lightweight, MIT license) embedded in React component
  • Data parsed from GANTT_Schedule_Master Google Sheet via Sheets API v4 on each sync cycle
  • SVG-based rendering for dependency arrows and milestone diamonds — crisp at all zoom levels
  • Virtualised row rendering for performance with 500+ activities
  • All date logic uses ISO 8601 (UTC) with WIB (+7) display conversion

Technology Stack & Tooling

Frontend

Framework: React (Next.js) — server-side rendering for performance in low-bandwidth environments.

Styling: Tailwind CSS — rapid, consistent UI with no heavy CSS frameworks.

Charts: Recharts or Apache ECharts — S-curves, heat maps, Gantt bars, donut charts.

Tables: TanStack Table (React Table v8) — sortable, filterable, paginated for MDR and procurement lists.

PDF Export: React-PDF or Puppeteer headless Chrome for weekly report generation.

Backend / Processing

Runtime: Node.js (Next.js API routes) — single codebase front+back.

Data connector: Google Sheets API v4 (googleapis npm) for Sheets; ExcelJS for parsing SharePoint-hosted Excel files.

Scheduling: GitHub Actions or Vercel Cron for automated sync every 6 hours.

Caching: In-memory (Redis optional) for latest dataset snapshot — enables offline/low-connectivity use.

Authentication & Access

Auth provider: Microsoft Azure AD (MS SSO) — aligns with Teams/SharePoint already in use by Adaro Group.

Fallback: Google OAuth 2.0 for users on Google Workspace.

Role management: NextAuth.js with custom role claims from Azure AD groups.

Hosting & DevOps

Hosting: Vercel (preferred for Next.js) or Azure App Service (if Adaro IT mandates Azure).

CI/CD: GitHub Actions — automated testing + deployment on merge to main.

Version control: GitHub private repository under PT Maruwai Coal organisation.

Monitoring: Vercel Analytics or Azure Monitor for uptime and performance tracking.

Alternative: Google AI Studio Integration

The Gemini API (Google AI Studio) can be embedded for natural language querying of the dashboard — e.g., "What are the top 3 overdue documents?" or "Summarise this month's commercial exposure." This is a Phase 2+ enhancement and does not block the core dashboard.

Development Environment

IDE: VS Code with ESLint + Prettier.

API testing: Postman or Bruno for Sheets API endpoints.

UI component library: shadcn/ui (pre-built accessible components).

Project management: GitHub Issues + Projects board for sprint tracking.

Module 13 — Gantt Chart: Baseline vs Actual Schedule Viewer

📅
This module provides a full interactive Gantt chart rendered directly in the browser, inspired by tools like OnlineGantt and MS Project's baseline view. It reads the master schedule WBS from Google Sheets and renders dual bars (Baseline vs Actual/Forecast), milestone diamonds, critical path, dependency links, and a live "Today" marker — with wet season risk bands specifically overlaid for the Barito River logistics window (Dec–Feb).

🗂️ WBS & Hierarchy

  • 4-level collapsible WBS tree (Project → Phase → Discipline → Activity)
  • Separate WBS tracks: EP Contract, Construction Contract, Milestones
  • Indent levels shown visually with expand/collapse toggles
  • Summary rows auto-calculate from child activities
  • Group-level bars span min–max of all children

📊 Dual-Bar Rendering

  • Baseline bar — thin grey bar below, from GANTT_Schedule_Master "Baseline Start / Finish"
  • Actual bar — solid coloured bar; colour coded by status: blue (planned), amber (in progress), green (complete), red (delayed)

⚙️ Implementation — Library Recommendation

The Gantt chart is the most technically demanding module. Recommended rendering libraries in order of preference:

1. DHTMLX Gantt (Community Edition) — most feature-complete open-source Gantt for the web. Supports baseline bars, dependencies, critical path natively. Free for open-source projects; commercial license ~$699/seat.

2. Frappe Gantt — lightweight open-source, beautiful default styling, supports dependencies. Best for milestone-focused views. Limited to simpler WBS hierarchies.

3. Custom D3.js Gantt — maximum flexibility; built from scratch using D3 scales and SVG. Higher development effort (~80 hrs) but perfectly tailored to CHPP2 data schema and branding.

Recommendation: DHTMLX Gantt for Phase 1 (speed to market), migrate to D3 custom in Phase 3 if needed.

📅 Wet Season Logic — Automated Flag Rules

The dashboard applies the following rules automatically when rendering the Gantt:

Rule 1 — Delivery in Wet Window: Any task in GANTT_Schedule_Master with Discipline = "Logistics" AND Forecast_Finish falls between 01-Dec–28-Feb → auto-flagged with ⚠ orange icon and added to the Logistics Risk Alert panel.

Rule 2 — Installation Delay: Any construction task with Baseline_Start in wet season + delayed status → flagged to PMC with recommended mitigation: "Consider pre-delivery storage at Puruk Cahu staging area."

Rule 3 — Milestone in Wet Season: Any milestone with Baseline_Finish in Dec–Feb → auto-highlighted in Milestone Tracker with a note about Barito River flood risk impacting access.

🔄 Data Sync & Baseline Lock

Baseline lock: Columns G–H (Baseline dates) are protected in the Google Sheet — only the Employer Admin can unlock. This prevents accidental overwriting of the approved baseline.

Sync frequency: Gantt data syncs every 6 hours via the standard sync engine. For real-time collaboration during progress meetings, a "Force Sync" button on the toolbar triggers an immediate pull.

Baseline revision: When the Employer formally revises the baseline (via VO/approved schedule update), a new baseline snapshot is saved with a timestamp. Users can toggle between "Current Baseline" and "Original Baseline" in the toolbar.

📱 Responsive & Performance

Large schedule handling: CHPP2 master schedule may contain 300–500+ activities. Virtual rendering (only render rows in viewport) keeps the chart responsive even on 500-row schedules.

Tablet mode: On screens ≤1024px, the task list collapses to show only Activity Name and %. Users can tap a row to expand the detail panel (dates, %, status) as an overlay.

Print mode: "Export PDF" triggers a full-page landscape render at A3 size, paginated by WBS group. Page headers include project name, export date, and "CONFIDENTIAL" watermark.

Offline mode: Last-synced Gantt data is cached in browser localStorage. If connection drops, the Gantt still renders with a "Last updated: [timestamp]" banner.
